embeddable analytics component

<maverick-analytics> is a self-contained web component (no framework, no CDN) served by this dashboard at /static/maverick-analytics.js. It reads the existing /api/v1/spend and /api/v1/goals endpoints and draws its charts as inline SVG. Below is a live instance running against this deployment:

using it on another page

<script src="http://YOUR-DASHBOARD:8765/static/maverick-analytics.js"></script>
<maverick-analytics
    endpoint="http://YOUR-DASHBOARD:8765"
    token="YOUR-MAVERICK_DASHBOARD_TOKEN"></maverick-analytics>

Honest limits

  • Same-origin requirement. The dashboard sends no CORS headers, so a page on a different origin cannot fetch its API — the browser blocks it. Embed on a page served by the dashboard itself (like this one) or behind the same reverse proxy / hostname.
  • Token exposure. A token attribute is visible to anyone who can read the embedding page's HTML, and it grants full dashboard API access. Only embed on pages whose audience you would trust with the dashboard itself. In no-token loopback mode, omit the attribute entirely.
  • Read-only data. The component only performs GETs against existing endpoints; it renders exactly what they return and shows the HTTP error when a fetch fails.
