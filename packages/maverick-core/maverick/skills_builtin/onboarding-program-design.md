# What this skill does

Designs an employee onboarding program for a specific role or cohort, producing a phased 30-60-90-day plan with milestones, owners, and readiness checkpoints. Output is a draft program a hiring manager or People team reviews and adapts per hire, not a fixed mandate.

# Steps

1. Gather the real inputs: role and level, team structure, manager, required systems/access, and role expectations. Use `knowledge_search` over the job description, role scorecard, team docs, and any existing onboarding templates; note anything assumed (e.g. access lists you couldn't confirm).
2. Lay out preboarding (offer-to-day-1): accounts, equipment, access requests, welcome comms, and first-week schedule — flag every access/provisioning item as an action a human must execute, never auto-grant.
3. Build the 30-60-90 plan. For each phase set a theme (30: learn/orient, 60: contribute, 90: own), 3-5 concrete milestones, the owner per milestone (manager / buddy / peer / new hire), and the evidence that proves it's met.
4. Add checkpoints (week-1, day-30, day-60, day-90 reviews) and a ramp/success definition. Report the full program, state assumptions, and hand off to the hiring manager and People team for approval before use.

# Notes

Output is wrong if milestones lack owners or evidence, or if access/provisioning is presented as done rather than as staged human actions. Do not invent system names, access scopes, or org policies — source them via `knowledge_search` or mark unverified. The skill drafts and recommends; account creation, access grants, and offer/comp details are irreversible/sensitive and stay with a human. Not for designing the broader L&D curriculum — use learning-program-design for that.
