# What this skill does

Prepares a calibration package for a performance review cycle so a calibration committee can compare proposed ratings consistently across managers. Produces a rating-distribution view, cross-manager and cross-cohort comparisons, and a flagged list of likely outliers and inconsistencies to discuss.

# Steps

1. Load proposed ratings with employee level, manager, team, and tenure; via knowledge_search pull the cycle's rating scale, definitions, and any target/guidance distribution so comparisons use the actual rubric, not assumptions.
2. In spreadsheet, build the overall and per-manager rating distributions and compare them to the guidance and to peer managers; compute how each manager's mix deviates (e.g. rating inflation, no low ratings).
3. Flag outliers for discussion — ratings inconsistent with stated evidence, managers far off distribution, similar performers rated differently across teams, and possible proximity/recency or leniency patterns; never silently change a rating.
4. Hand off the package with distributions, deviation metrics, and a discussion-flag list, stating that flags are prompts not verdicts; final ratings are decided by the calibration committee, not adjusted automatically.

# Notes

Output is wrong if a guidance distribution is treated as a hard quota (forced ranking unless explicitly the policy), if cohorts are too small to read a distribution into, or if flags are framed as conclusions. Watch for surfacing protected-attribute patterns — that is a fairness signal to raise with HR, not to act on unilaterally. Do not use to mechanically fit ratings to a curve; calibration adjusts inconsistencies, and every change stays a human committee decision.
