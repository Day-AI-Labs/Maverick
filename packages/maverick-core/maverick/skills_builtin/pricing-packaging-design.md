# What this skill does

Designs pricing tiers and packaging for a product line: groups features into tiers along
a value metric, sets the fences that separate tiers, and identifies add-ons. Produces a
packaging design — tier table, fences, value metric, and add-on list — grounded in
willingness-to-pay evidence and competitive benchmarks, with the unit economics modeled.

# Steps

1. Gather inputs with `knowledge_search`: feature inventory, target segments and their
   jobs, competitor packaging and price points, any willingness-to-pay or win/loss data,
   and cost-to-serve per feature. Mark missing economics or WTP data as unverified.
2. Choose the value metric — the unit that scales with the customer's realized value and
   that they understand (seats, usage, records, etc.) — and confirm it is measurable and
   billable. The metric anchors the whole structure, so pick it before the tiers.
3. Group features into a good/better/best ladder: put broadly-needed capability in the
   entry tier and segment-specific or high-WTP capability above. Define explicit fences
   (the capability or limit that makes a buyer step up) so tiers do not cannibalize.
4. Model the tiers in a `spreadsheet`: list price per tier, expected mix, ARPU, margin
   given cost-to-serve, and the add-ons priced separately. Report the packaging design and
   the model, state assumptions (WTP, mix, costs), and hand off to a pricing owner; do not
   publish prices.

# Notes

The design is wrong if fences are weak (everything important sits in the entry tier, so
no one upgrades) or if the value metric does not track value (flat per-seat pricing on a
usage-driven product strands margin). Benchmarks and WTP figures must be cited; do not
fabricate competitor prices — mark them unverified if not confirmed. Run the margin model
before recommending any list price. This drafts packaging for a human pricing owner —
list prices, discount policy, and any change to live pricing are revenue-affecting and
require their approval. Do not use to negotiate a single deal's discount or to model an
existing customer's renewal (those are deal-desk tasks, not packaging design).
