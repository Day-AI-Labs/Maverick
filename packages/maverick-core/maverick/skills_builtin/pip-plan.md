# What this skill does

Structures a performance-improvement plan (PIP) for an employee whose performance is below expectation, translating vague concerns into specific, measurable goals with support and a review cadence. Produces a draft PIP with SMART goals, manager support commitments, milestones, and scheduled check-ins for HR and manager review.

# Steps

1. Gather the specific documented performance gaps (with dates/examples), the role's expectations, and prior feedback already given; via knowledge_search pull the org's PIP policy, required duration, and templates so the plan conforms — do not invent timelines or escalation language.
2. Convert each gap into a SMART goal: observable behavior or output, a measurable threshold tied to the role standard, and a target date inside the policy window; avoid goals that are subjective or that can't be evidenced.
3. For each goal define the support the manager/org provides (training, mentoring, resources, cadence of feedback) and the milestones with explicit dates; set review check-ins (e.g. weekly) and a clear final review date.
4. Hand off the DRAFT PIP for manager, HR, and where required legal review, stating assumptions and noting any gap lacking documented evidence; the decision to issue the PIP and any consequence of it are human and irreversible — never finalize or communicate it autonomously.

# Notes

Output is wrong if goals are unmeasurable, if the gaps cited aren't backed by documented evidence, if the timeline violates policy/contract, or if it reads as predetermined termination rather than genuine support. A PIP can carry legal and employment-law weight — keep it factual, behavior-based, and free of conclusions about the person. Do not use for routine coaching or for misconduct (a different disciplinary process); always route through HR before delivery.
