# What this skill does

Condenses a longer artifact (report, analysis, proposal, incident review) into a one-page executive summary that leads with the decision-relevant conclusion. Produces a summary where the "so-what" — the recommendation or headline finding — appears in the first two sentences, followed by the minimum supporting evidence a busy executive needs to act.

# Steps

1. Pull the source material with `knowledge_search`: the document(s) being summarized plus any prior decisions, targets, or constraints that frame what "matters" to this audience. Identify the single decision or question the reader cares about.
2. Draft the lead: state the bottom-line conclusion or recommendation up front (the so-what), then 3-5 supporting points — each a claim with a number or evidence pointer, not a process narrative.
3. Add a short "what we need from you" line if a decision, approval, or resource is being requested; omit it if the summary is purely informational.
4. Cite each load-bearing figure back to its source section; mark any number you could not verify as `[unverified]`. Hand off the one-pager, stating which claims came from the source versus your synthesis.

# Notes

The output is wrong if it buries the conclusion under background, exceeds one page, or includes any figure not traceable to the source. Do not fabricate metrics to fill a section — an honest gap ("impact not yet quantified") beats an invented number. This is a drafting aid: a human owns the final wording and any commitment it implies. Not for cases where the reader needs the full reasoning chain — write a report, not a summary, then.
