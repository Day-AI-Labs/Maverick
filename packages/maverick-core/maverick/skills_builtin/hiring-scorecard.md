# What this skill does

Defines a structured hiring scorecard for a specific role so interviewers evaluate candidates consistently and defensibly. Produces a scorecard listing weighted competencies, the observable signals (and red flags) for each, and behavioral rating anchors (e.g., 1-4) so scores map to evidence, not gut feel.

# Steps

1. Gather inputs: the job description, the role's must-have vs. nice-to-have requirements, and the interview-loop structure. Use knowledge_search to pull the role profile, leveling guide, and any existing scorecards to align with company standards.
2. Derive 4-7 core competencies (a mix of role-specific skills, collaboration, and leveling expectations). Assign each a weight reflecting importance; tie every competency to a requirement from the JD so none is invented.
3. For each competency write observable signals (what a strong answer/work sample looks like) and red flags, then define rating anchors for each point on the scale so "3" means the same thing across interviewers.
4. Map competencies to interview stages so coverage has no gaps or redundancy, and add an explicit guard against unstructured/biased criteria (no "culture fit" without behavioral definition). Report the scorecard, flagging any competency lacking a JD basis as "to confirm with hiring manager."

# Notes

Output is wrong if anchors are vague ("good communicator" with no observable behavior), if weights don't reflect the JD, or if it includes legally risky or proxy criteria. A scorecard standardizes judgment; it does not make the decision — the hiring manager and panel own offer calls, and any adverse-impact concern goes to HR. Do not reuse a scorecard across materially different roles or levels without re-deriving competencies.
