# What this skill does

Structures a new hire's first 90 days into a phased 30-60-90 plan: learn (30), contribute (60), own (90). Produces a plan with concrete milestones, named stakeholders, and measurable success criteria per phase, grounded in the role's actual scope rather than generic templates.

# Steps

1. Gather inputs: role title, level, team, manager, and the role's charter or job description. Use knowledge_search to pull the existing role profile, team OKRs, and any onboarding docs; note what is missing rather than guessing.
2. Define the 30-day phase (learn): orientation, systems/access, key relationships to build, and 2-3 ramp milestones with a clear "done" signal (e.g., shipped first small change, completed required training).
3. Define the 60-day phase (contribute): first owned deliverable, dependencies, and the manager checkpoint. Tie each milestone to a measurable outcome drawn from team OKRs found in step 1.
4. Define the 90-day phase (own): full-scope ownership, a stretch goal, and the success measures the manager will use to evaluate ramp. Report the plan with assumptions flagged (any phase built on missing inputs marked "to confirm with manager").

# Notes

Output is wrong if milestones are generic ("get up to speed") instead of role-specific and measurable, or if success measures are not tied to real team goals. Mark any milestone derived from absent docs as unverified. This is a draft plan: the hiring manager owns final targets and any commitment that affects compensation or formal review. Do not use for performance-improvement plans (PIPs) — those are a separate, higher-stakes process.
