# What this skill does

Designs a structured customer onboarding journey from signup to first value and steady-state adoption. Produces a milestone map with entry/exit criteria, owner, and customer-facing actions per stage, plus measurable success criteria (notably time-to-value). This is a journey blueprint, not an automated workflow.

# Steps

1. Ground the design in real context via `knowledge_search`: pull the product's core "aha" / first-value moment, the standard activation steps, the target customer segment, and any existing onboarding docs or SLAs. If the first-value moment is undefined, define it explicitly with the team before sequencing — everything anchors to it.
2. Sequence the journey into milestones (e.g., kickoff, technical setup, first-value, adoption, handoff to steady state). For each, write entry criteria, exit criteria, the customer-facing action, and the responsible owner (CSM, onboarding specialist, product).
3. Attach success criteria per milestone — a leading metric and a target — and define time-to-value as the elapsed time from start to the first-value exit criterion. Add risk checkpoints where stalls commonly occur (e.g., setup blocked, no data loaded) with an escalation path.
4. Deliver the journey as a milestone table plus a one-paragraph narrative of the happy path. Hand off to the onboarding owner, stating which segment it targets, which metrics already exist vs. need instrumentation, and noting it should be validated against a real cohort before rollout.

# Notes

Wrong if milestones lack measurable exit criteria (then "done" is subjective), if time-to-value isn't tied to a concrete first-value event, or if the design ignores the actual product's activation path (a generic journey that doesn't fit the tool fails on contact). Don't invent metrics that aren't instrumented — mark them as needing instrumentation. This is a recommended blueprint; a human owns rollout and any customer-facing automation. Do NOT use it for a single bespoke enterprise implementation that needs a custom project plan, and do NOT skip the segment scoping — one journey rarely fits self-serve and enterprise both.
