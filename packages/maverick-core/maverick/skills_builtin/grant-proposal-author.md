# What this skill does

Drafts a complete grant proposal tailored to a specific funder and program, aligning the applicant's project to the funder's priorities and required sections. Produces a proposal with statement of need, project plan, budget, and measurable outcomes ready for program-staff review.

# Steps

1. Use `knowledge_search` to retrieve the funder's RFP/guidelines (eligibility, funding range, required sections, deadline, evaluation criteria) and the applicant's own materials (mission, prior outcomes, financials, the project concept). Cite each source; mark anything not found as unverified rather than assuming it.
2. Write the statement of need grounded in cited data (population, problem magnitude, gap), then a project plan with goals, activities, timeline, and staffing that directly maps to the funder's stated priorities and evaluation criteria.
3. Build a budget and justification consistent with the funder's funding range and allowable-cost rules, tying each line to a project activity; flag any cost category the guidelines restrict. State measurable outcomes with baselines, targets, and an evaluation method.
4. Assemble the full proposal in the funder's required section order and within stated length/format limits. Report any missing inputs or eligibility risks, list the citations used, and hand off to a program officer for review and submission.

# Notes

Output is wrong if it fabricates statistics, outcomes, or organizational track record, misstates eligibility, or ignores the funder's required structure, page limits, or deadline — any of these can disqualify the application. Never invent need data or letters of support; mark gaps for a human to supply. This skill produces a draft only — a program officer or authorized signatory reviews and submits; submission is irreversible and binding. Do not use for non-grant fundraising (individual appeals, sponsorships) or for post-award reporting.
