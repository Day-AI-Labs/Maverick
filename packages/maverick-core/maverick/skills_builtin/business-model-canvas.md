# What this skill does

Builds a Business Model Canvas for a company, product, or venture across the nine building blocks: customer segments, value propositions, channels, customer relationships, revenue streams, key resources, key activities, key partnerships, and cost structure. Produces a populated canvas plus a short read on coherence — where the blocks reinforce or contradict each other. Each block is grounded in known facts about the entity, not invented.

# Steps

1. Identify the entity and scope: the specific business, product line, or venture being modeled, and whether the canvas is as-is (today) or to-be (proposed). State which.
2. Populate the demand side using `knowledge_search` for what is known about the entity: customer segments, value propositions, channels, customer relationships, and revenue streams. Cite the source for each entry; mark inferred entries as `[unverified]`.
3. Populate the supply side: key resources, key activities, key partnerships, and cost structure. Tie each cost and activity back to the value propositions it serves.
4. Check coherence: confirm each value proposition reaches a named segment through a channel and is paid for by a revenue stream, and that resources/activities/partners actually produce it. Flag any orphaned or contradictory block.
5. Report: deliver the nine-block canvas plus the coherence notes and the list of `[unverified]` entries. Restate whether the canvas is as-is or to-be and which blocks are thinnest on evidence.

# Notes

The canvas is wrong when blocks are filled with plausible-sounding placeholders instead of entity-specific facts, when revenue streams don't map to value propositions, or when an as-to-be model is presented as the current state. This is a descriptive/strategy-shaping artifact, not a commitment — proposed model changes are recommendations a human decides on. Do not use to evaluate entering a new market (use market-entry-analysis) or to rank rivals (use competitive-landscape-map); needs a defined entity to model.
