# What this skill does

Produces a structured job description for a single role: a titled JD with sections for mission/summary, responsibilities, must-have and nice-to-have requirements, and the level/grade it maps to. Output is a hiring-ready draft grounded in the org's existing leveling rubric and comparable roles, not a generic template.

# Steps

1. Establish the role inputs: title, function/team, target level, and reporting line. If any are missing, knowledge_search the org's leveling guide and adjacent JDs to infer a default and flag it as an assumption rather than inventing it.
2. knowledge_search for the relevant leveling rubric and 2-3 comparable existing JDs in the same family; extract the scope/impact/autonomy expectations for the target level so responsibilities are calibrated, not aspirational.
3. Draft the JD: one-line mission, 4-7 outcome-oriented responsibilities (what they own, not tasks), requirements split into must-have vs nice-to-have, and an explicit level mapping with one sentence justifying it against the rubric.
4. Hand off the JD as a draft, listing every assumption made (inferred level, comp band omitted, location/remote unstated) and noting that level assignment and posting are a human decision.

# Notes

The JD is wrong if responsibilities are copied from a higher or lower level than the stated target, if must-haves include screen-out criteria that aren't genuinely required (a fairness and legal risk), or if any requirement is fabricated rather than drawn from a comparable role. Cite the comparable JDs and rubric used; mark anything inferred. Do not invent compensation, headcount, or EEO/legal language — stage those for HR. Not for re-leveling an existing employee or for sourcing/outreach copy.
