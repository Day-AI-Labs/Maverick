# What this skill does

Builds a benefits realization plan that defines how a program's expected value will be measured, owned, and reviewed over time. Produces a structured benefits register with metrics, baselines, targets, owners, and a tracking cadence so claimed value is verifiable rather than asserted.

# Steps

1. Gather the program's stated objectives and the benefits hypothesis (financial and non-financial). For each benefit, capture its source — business case, sponsor statement, or estimate — and flag any benefit with no documented basis as unverified.
2. For every benefit, define one measurable metric, its current baseline (with the baseline date and data source), a target, and the realization timeframe. If a baseline cannot be sourced, record "baseline TBD" rather than inventing a number.
3. Assign a single accountable owner per benefit and set a review cadence (e.g. monthly leading indicators, quarterly value review). Note dependencies and the conditions that would invalidate a benefit.
4. Build the plan in `spreadsheet`: a benefits register (benefit, metric, baseline, target, owner, cadence, status) plus a realization timeline. Report the plan, list unverified benefits and missing baselines, and hand the register to the program owner for sign-off.

# Notes

The plan is wrong if benefits lack a measurable metric, if baselines are fabricated rather than sourced, or if a benefit has no single accountable owner (shared ownership = no ownership). This is a tracking design, not a value guarantee: do not assert benefits were realized — that is confirmed later against actuals by the owner. Skip if there is no business case or objective to anchor to; establish one first.
