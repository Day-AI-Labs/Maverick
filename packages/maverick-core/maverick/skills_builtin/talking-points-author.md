# What this skill does

Prepares a spokesperson briefing for an interview, panel, or press moment: a tight set of key messages plus bridges that pivot from hard or off-topic questions back to those messages. Produces a one-page, grounded talking-points doc the spokesperson can internalize, every claim sourced.

# Steps

1. Confirm the audience, format, spokesperson, and the 2-4 outcomes they want the audience to remember. Without a stated objective, ask before drafting — generic talking points are useless.
2. Run `knowledge_search` to ground each key message in verifiable facts (metrics, announcements, prior public statements); capture sources and flag anything UNVERIFIED or not-yet-public.
3. Draft 3-5 key messages, each one sentence the spokesperson can say verbatim, plus a one-line supporting proof point per message.
4. For each anticipated hard or off-topic question, write a bridge ("That's important, and what matters here is...") that redirects to a key message. Hand off the doc, list questions you could not safely bridge, and flag any claim not yet cleared for public use.

# Notes

The output is wrong if it puts unverified, confidential, or not-yet-announced facts in the spokesperson's mouth — mark those explicitly and exclude from the speakable set. Bridges must redirect, not dodge in a way that reads as evasive. This is a draft for the spokesperson and comms lead to review and approve; never treat it as cleared messaging. Do not use for internal-only notes or for legally sensitive statements that require counsel.
