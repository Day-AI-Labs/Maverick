# What this skill does

Identifies the stakeholders of an initiative, scores each on influence and interest, and produces an engagement plan keyed to the four quadrants (manage closely / keep satisfied / keep informed / monitor). Output is a stakeholder map plus per-stakeholder actions, owners, and cadence.

# Steps

1. Capture the initiative, its scope, and any named stakeholders from the requester. Run `knowledge_search` for org charts, prior project records, and meeting notes to surface additional stakeholders; cite each source.
2. For each stakeholder, assign influence (high/low) and interest (high/low) with a one-line rationale. Mark any score based on inference rather than a cited signal as `[assumed]`.
3. Place each into the influence/interest grid and derive the quadrant strategy: manage closely, keep satisfied, keep informed, or monitor.
4. Produce an engagement plan (action, owner, cadence per stakeholder) and hand off, stating that scores are assessments to be validated with the project sponsor.

# Notes

Output is wrong if scores are asserted without rationale, or if power-brokers are missed because only the obvious participants were listed — the knowledge_search step exists to widen the net. Do not record private/sensitive judgments about individuals as fact; mark assessments as assumptions. This drafts a plan; the sponsor approves any outreach. Not for one-off comms or initiatives with a single stakeholder.
