# What this skill does

Designs a channel/partner program: a tiered structure (e.g. Registered/Silver/Gold) with named entry criteria, an enablement path per tier, and an incentive model. Produces a recommendation document an RVP/channel lead can review — tier definitions, qualification thresholds, partner-facing benefits, vendor-side margin/MDF/rebate mechanics, and an enablement and certification ladder. Output is a draft proposal, not an activated program.

# Steps

1. Pull context with knowledge_search: existing partner agreements, current discount/margin schedules, product lines partners can sell, target segments/geos, and any prior program docs. Record what was found vs. what is missing — do not invent partner counts, deal economics, or competitor terms.
2. Define tiers from real anchors: set 3-4 tiers with explicit, measurable entry criteria (annual sourced/influenced revenue, # certified reps, # net-new logos). Tie thresholds to the actual revenue distribution found in step 1, not round numbers pulled from air.
3. Build the enablement path per tier: required certifications, sales/technical training, demo/lab access, and co-marketing eligibility. Map each benefit (deal registration protection, lead share, MDF %, rebate %, margin uplift) to the tier that earns it, and check the total give-away against the margin schedule from step 1 for affordability.
4. Assemble the incentives: front-end margin, back-end rebates on attainment, MDF tied to pipeline, and deal-registration protection windows. Flag every assumed number (rebate %, MDF pool size) as UNVERIFIED where source data was absent.
5. Report the program as a structured draft: tier table, enablement ladder, incentive matrix, and an open-questions list. State assumptions explicitly and hand off for finance/legal sign-off before any partner-facing commitment.

# Notes

Output is wrong if tier thresholds ignore the actual partner revenue distribution (tiers no one can reach, or everyone lands in the top tier), or if stacked incentives exceed available margin — always reconcile the incentive matrix against the real discount schedule. Margin, rebate, and MDF commitments are contractual and irreversible once published: this skill only drafts and recommends; finance and legal approve, and a human launches. Do not use for a single one-off reseller agreement (that's a deal desk task, not a program), and do not cite numbers as facts when the underlying agreements weren't found in knowledge_search — mark them UNVERIFIED.
