# What this skill does

Plans a publishable editorial calendar for a defined window (e.g. a quarter): a dated list of content pieces with topic, format, channel, owner, and publish date, derived from real strategy inputs rather than invented themes. Produces a calendar table ready for a human editor to approve and load into a planning tool.

# Steps

1. Pull the inputs with `knowledge_search`: the content strategy, audience segments, product launches, key dates/seasonality, and any existing backlog or past performance. Note the cadence (posts/week) and channels actually in scope.
2. Derive themes from those inputs — map each segment and launch to topic pillars. Do not introduce topics with no source; mark any gap as "[needs input]" rather than filling it.
3. Lay out the timeline: assign each piece a publish date honoring cadence and known key dates, a format (blog/video/email/social), a channel, and an owner. Flag dependencies (e.g. content gated on an unreleased feature).
4. Output the calendar as a dated table (date, topic, pillar, format, channel, owner, status). Report assumptions (cadence, window, owner availability) and hand off for editor approval.

# Notes

Output is wrong if dates ignore known launches/seasonality, if owners are assigned without confirming availability, or if topics are fabricated beyond the strategy inputs. The calendar is a draft recommendation — a human editor owns final scheduling and resourcing. Don't use this for ad-hoc single-piece briefs or for reactive/newsjacking content where dates can't be planned ahead.
