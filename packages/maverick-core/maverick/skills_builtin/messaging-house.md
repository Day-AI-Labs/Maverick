# What this skill does

Builds a messaging house for a product or brand: a single overarching value proposition (the "roof"), 3-4 supporting message pillars, the proof points that substantiate each pillar, and a one-paragraph elevator pitch. Produces a reusable framework that sales, marketing, and product can all draw from with one voice.

# Steps

1. Gather the raw inputs with `knowledge_search`: existing positioning docs, the target audience/ICP, the differentiated capabilities, competitor framing, and any customer-quoted outcomes. Note what exists vs. what is missing — do not invent audience pain or differentiators.
2. Draft the roof: one value proposition stating who it's for, what category it's in, and the single most important benefit. Keep it to one sentence; if it needs two, the positioning isn't decided yet — flag that.
3. Define 3-4 pillars, each a distinct reason-to-believe that ladders up to the roof (not overlapping). Under each pillar, attach proof points — features, metrics, customer outcomes, certifications. Mark any proof point you could not source as [UNVERIFIED] rather than asserting it.
4. Write the elevator pitch (3-4 sentences) synthesizing roof + pillars in plain language. Hand off the full house, listing which proof points are sourced vs. unverified and which audience assumptions need a human to confirm.

# Notes

The framework is wrong when pillars overlap (they should be orthogonal), when the roof is a feature list instead of a benefit, or when proof points are aspirational claims dressed as facts — those expose the brand to credibility and compliance risk, so they must be marked unverified and left for a human to validate. This drafts and recommends; it does not publish to a website, deck, or ad. Do not use it for a single campaign tagline or a one-off email — that's copywriting, not a framework.
