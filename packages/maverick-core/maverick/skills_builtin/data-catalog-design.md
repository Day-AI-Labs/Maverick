# What this skill does

Designs a data catalog: the metadata model, ownership and certification, and the search/lineage experience that makes data findable and trustworthy.

# Steps

1. Define the metadata model: business and technical metadata, glossary terms, ownership, certification status, and sensitivity.
2. Specify population: automated harvesting from sources plus the human curation (descriptions, certification) that automation can't supply.
3. Design discovery: search, browse by domain, lineage, and the trust signals (certified, deprecated, freshness) users need to pick the right asset.
4. Define the operating model for stewardship and freshness. State assumptions and hand off.

# Notes

Catalogs rot when nothing keeps metadata fresh and nobody certifies assets — search then returns junk. Curation and trust signals are the point. Tool selection is a separate decision this skill informs.
