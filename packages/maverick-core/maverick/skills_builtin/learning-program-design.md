# What this skill does

Designs a structured learning/training program for a defined audience and skill gap, producing measurable objectives, a sequenced module outline, and an assessment plan. Output is a draft program design a human L&D owner reviews before rollout, not an auto-published curriculum.

# Steps

1. Pull the real inputs: target audience, current vs. desired competencies, time/budget constraints, and any existing role profiles. Use `knowledge_search` over internal competency frameworks, job descriptions, and prior training materials; mark any gap you had to assume rather than source.
2. Write learning objectives as observable, measurable outcomes (Bloom-style verbs: "configure", "diagnose", not "understand"), each tied to a named competency gap.
3. Sequence modules from foundational to applied, scaffolding prerequisites first. For each module record: objective(s) covered, format (instructor-led / self-paced / on-the-job), duration, and required materials — cite sourced content; flag content that must be authored.
4. Define assessment per objective (knowledge check, practical task, manager sign-off) plus an overall completion bar and a feedback/iteration loop. Report the full design, list every assumption, and hand off to the L&D owner for approval before scheduling.

# Notes

Output is wrong if objectives are unmeasurable, modules are unsequenced, or assessments don't map back to objectives. Do not fabricate competency frameworks or compliance requirements — if not found via `knowledge_search`, label them unverified and request the source. The skill drafts and recommends; a human owns scheduling, vendor spend, and any mandatory/compliance certification. Not for one-off lunch-and-learns or single job-aids — use only when a multi-module program with assessment is warranted.
