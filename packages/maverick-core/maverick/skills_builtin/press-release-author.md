# What this skill does

Drafts a standard-format press release for a specific announcement (launch, funding, partnership, milestone). Produces a ready-to-review release with dateline, headline and subhead, inverted-pyramid body, attributed quotes, boilerplate, and media contact block.

# Steps

1. Gather the facts via knowledge_search: what/who/when/where/why, approved numbers, official names and titles, the launch date, and the company boilerplate. Use only confirmed facts — never invent metrics, dates, or quotes.
2. Write a factual headline and one-line subhead leading with the most newsworthy point; draft the lead paragraph to answer the core who/what/when/where/why up front.
3. Build the body in inverted-pyramid order: supporting detail, context, and proof points, each traceable to a sourced fact. Flag any gap where a fact is missing rather than filling it.
4. Draft one or two quotes attributed to named spokespeople, written in their voice but marked DRAFT — quotes require sign-off before publication. Append the approved boilerplate and a media-contact placeholder.
5. Report the full release plus a list of items needing confirmation (quote approvals, embargo date, final figures) and any assumption made.

# Notes

The release is wrong if it contains an unverified number, a fabricated quote, or a date not confirmed by the source. All quotes are drafts pending spokesperson approval; publication and distribution are human-controlled, irreversible actions — never auto-send. Skip when the announcement isn't approved or the facts aren't locked; a release built on placeholders is not shippable.
