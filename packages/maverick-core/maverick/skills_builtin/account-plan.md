# What this skill does

Builds a strategic account plan for a named key or target account: a structured view of current footprint, whitespace, the stakeholder map, and prioritized plays to expand revenue. Produces a single account-plan document that a rep or AE can act on and a manager can review. It plans growth; it does not commit pricing, contracts, or external outreach.

# Steps

1. Pull everything known about the account via `knowledge_search`: existing contracts/products owned, ARR, segment, industry, renewal dates, open opportunities, support/health signals, and prior plan notes. Record the account name and as-of date; mark any field as unverified if no source backs it.
2. Map the current footprint against the full sellable catalog to surface WHITESPACE — products/modules/use-cases the account does NOT yet own but peers in its segment do. Cite the comparison basis (segment benchmark or catalog) rather than guessing fit.
3. Build the stakeholder map: economic buyer, champions, technical evaluators, and blockers, with each person's role, sentiment (supporter / neutral / detractor), and last-touch date from the retrieved records. Flag coverage gaps (e.g., no identified economic buyer).
4. Define 3-5 prioritized plays, each as {objective, target whitespace, entry stakeholder, trigger/timing, first action, rough value}. Order by value-over-effort. Report the plan stating assumptions (e.g., renewal date, ARR figures) and which fields were unverified; hand off for human review before any outreach.

# Notes

Wrong if it invents owned products, ARR, or stakeholders not present in the source — every claim must trace to a retrieved record or be labeled unverified. Whitespace is only credible when the peer/catalog comparison basis is stated. This is a planning artifact: it stages plays and target contacts but does NOT send outreach, quote prices, or alter the CRM — a human owns those. Do not use for net-new prospecting (no footprint to plan against) or for a pure renewal forecast.
