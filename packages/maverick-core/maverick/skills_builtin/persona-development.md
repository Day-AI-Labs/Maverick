# What this skill does

Defines a small set of evidence-based buyer or user personas for a product or market: each with role/context, goals, pains, buying or usage triggers, decision criteria, common objections, and where they get information. Produces persona profiles grounded in internal research and external signal, with every attribute traceable to a source and gaps marked as assumptions. Output informs messaging and targeting; it is not a finalized GTM decision.

# Steps

1. Clarify scope: which product/market, buyer vs. user personas, and how many (typically 2-4 distinct, non-overlapping). Identify the differentiating axis (role, company size, use case, maturity) so personas are genuinely separable rather than cosmetic variants.
2. Gather internal evidence with `knowledge_search` — win/loss notes, sales-call summaries, support themes, existing research, CRM segment data — and tag findings to candidate personas. Use `web_search` to corroborate or fill external context (role responsibilities, industry pressures, common objections), citing each source and dating anything time-sensitive.
3. Synthesize each persona: role and context, top goals, key pains, triggers that start a buying/usage cycle, decision criteria and influencers, likely objections, and trusted information channels. Where evidence is thin, write the attribute as an explicit hypothesis to be validated, not as fact.
4. Deliver the persona set with per-attribute citations, a clear list of assumptions/validation gaps, and a one-line statement of how the personas differ from each other. Hand off as input for messaging, targeting, and content; positioning and spend decisions stay with marketing/product.

# Notes

Personas are wrong when they are demographic fan-fiction — invented names and stock motivations with no source; ground every attribute or flag it unverified, and never fabricate quotes or stats. Overlapping personas that don't drive different decisions are noise; if two can't be told apart by goals/pains/triggers, merge them. Distinguish corroborated internal evidence from single-source web claims, and date anything that ages. These are recommendations to validate with real buyers, not settled truth. Do not use when zero customer/market evidence exists — that yields speculation, not personas.
