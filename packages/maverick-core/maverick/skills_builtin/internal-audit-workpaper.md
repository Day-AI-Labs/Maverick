# What this skill does

Structures an internal-audit workpaper for a single test step so the work is review-ready and self-supporting. Handles the goal class "document one audit procedure end to end" and produces a workpaper with objective, procedure performed, evidence/sample detail, exceptions, and a conclusion that ties back to the objective. The output is the standard four-part record an audit reviewer expects to sign off.

# Steps

1. Anchor the objective to the audit program / risk-and-control matrix via knowledge_search: which risk or control this step addresses and what "satisfied" looks like. Quote the program reference (step ID); if no program step maps, flag it — a workpaper without a stated objective is not reviewable.
2. Document the procedure as actually performed: population source and size, selection basis, period covered, and the steps run. Reference (not paraphrase) the underlying evidence — file names, screenshots, query IDs — so a reviewer can re-trace. Mark anything not independently verified as unverified.
3. Record results at the sample level: what was inspected, pass/exception per item, and a description of each exception with its tickmark. Keep raw evidence separate and cross-referenced; the workpaper narrates, the evidence proves.
4. Write a conclusion that explicitly answers the objective (objective met / met with exceptions / not met) and lists open items or proposed findings. Report assumptions and any scope limitation. Hand off for reviewer sign-off; do not finalize the rating or issue a finding — that is the reviewer/lead's decision.

# Notes

Output is wrong if the conclusion doesn't address the stated objective, if evidence is described but not referenced (un-re-performable), or if exceptions are buried in narrative instead of itemized. A workpaper asserting a conclusion without traceable evidence fails review. This skill drafts and structures; the finding, its rating, and report inclusion are human decisions. Do not use to draft the audit report itself or to design the overall audit program — this is one test step.
