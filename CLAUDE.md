## Rules

### Environment

- Run tests as `python3 -m pytest`, never bare `pytest`: PATH `pytest` can be
  a uv-tool/pipx shim whose isolated venv cannot see the installed packages
  (observed: 173 collection errors from the shim, 0 from `python3 -m`).
- A `pyo3_runtime.PanicException` at collection hides the real error on the
  captured-stderr line (here `No module named '_cffi_backend'`: distro
  cryptography needs cffi — post-create.sh now installs it).
- pytest aborts the WHOLE run on any collection error — 5 broken imports kept
  9000 green tests from running. Fix imports before judging the suite.
- `pip install -e .` at repo root fails: the root pyproject.toml is uv-workspace
  + tool config only (no `[project]`). Install per-package.
- `python3 -m build` without pypa-build installed produces a misleading
  "'build' is a package" error because the repo has a `build/` dir
  (PyInstaller spec). It is missing-dependency, not a real conflict.
- `ANTHROPIC_API_KEY` and `~/.maverick/config.toml` are absent in fresh
  environments: live-LLM paths are untestable; `maverick doctor` reports it;
  `/healthz` returns 503 by design (deep check: no provider key = degraded).
  mkdocs is also not installed by post-create.sh.

### Testing

- Tests live at `packages/<pkg>/tests/test_<module>.py`; the module is the
  unit (e.g. `money_format.py` -> `test_money_format.py`). Tool wrappers under
  `tools/` often have NO standalone test file — grep before assuming a name.
- Dashboard tests: `TestClient(app, headers={"Origin": "http://testserver"})`
  — mutating /api/v1 requests in no-token mode must pass the same-origin CSRF
  check — plus an autouse fixture monkeypatching `world_model.DEFAULT_DB` to
  `tmp_path` so tests never touch the real world DB. Copy that pattern.
- `Budget` enforces ALL caps inside `record_tokens` (tokens, $, wall, tools),
  not just at `check()`: a test raising only `max_dollars` still dies on the
  default `max_output_tokens=200_000`. Raise every cap you'll cross.
- pytest-asyncio runs in STRICT mode; async tests need explicit markers.

### Lint / CI gates beyond ruff

All verified locally green; each is a build-failer:
- Bare `import tomllib` at column 0 (greps apps/packages/benchmarks). Use:
  `try: import tomllib` / `except ModuleNotFoundError: import tomli as tomllib`
  (3.10 support; this broke three PRs).
- `shell=True` outside `maverick/sandbox/` (route shell through `sandbox.exec()`).
- detect-secrets vs `.secrets.baseline` (new hashes fail; `pragma: allowlist
  secret` for false positives).
- `python -m maverick.plugin_matrix --ci`, `python -m maverick.deprecations
  --ci` (past-due deprecations fail), `python -m maverick.grpc_api.contract
  --check` (proto removals/renumbers fail; additive only), `python -m
  maverick.a11y_audit --ci` (dashboard templates).
- Bugbear (B) is OFF on purpose: 127 latent findings (61 B904, 45 B905...).
  Do not fix or enable them in an unrelated PR. Complexity capped at 20 with
  30 grandfathered `# noqa: C901` — new code stays under the cap.
- No mypy/pyright anywhere. Ruff is the only static gate; don't add type-only
  "fixes" expecting a checker to validate them.

### File structure / imports

- Inside a package use relative imports (`from ..money_format import ...`);
  tests use absolute (`from maverick.budget import ...`).
- A module rename must also update `docs/FEATURES.md` (it references module
  names; a code-only rename leaves it stale) and any `tools/` wrapper.
- Reverting a `git mv` via `git checkout -- .` leaves the new-name file behind
  as untracked; follow with `git clean` or rm.

### Common mistakes

- Do NOT add a health endpoint under `/api/v1` — `/healthz`, `/livez`,
  `/readyz` exist on the app, deliberately auth-exempt (LB probes can't carry
  a bearer; payload redacts when MAVERICK_DASHBOARD_TOKEN is set).
- Do NOT trust `cmd | tail` exit codes — that's tail's status (it masked a
  pytest collection abort here; use PIPESTATUS or a log file). Shell cwd also
  persists between steps — anchor pytest invocations at the repo root.
- New capabilities need a config knob AND a wizard step in
  `apps/installer-cli/maverick_installer/` (rules 5-6 below).

### Dependencies (live docs, June 2026)

- `anthropic` floor `>=0.40.0`; current SDK is v0.109.1. `client.messages.create`
  pattern unchanged; pin floors when touching provider code.
- FastAPI: `@app.on_event` is deprecated (lifespan handlers replace it).
  `app.py:285` still uses it — the suite's DeprecationWarnings are known noise;
  don't "fix" drive-by, it's an intentional pending migration.
- MCP protocol current = `2025-11-25` — exactly what maverick-mcp implements
  (plus `2024-11-05` fallback). Don't bump without the spec. (Also expect
  starlette's "install httpx2" TestClient deprecation warning — known noise.)
